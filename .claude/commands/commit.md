현재까지의 변경 사항을 분석하여 하나의 Git 커밋을 생성하라. 커밋 메시지는 다음 규칙을 따른다.

- 커밋 메시지는 한국어로 작성한다.
- 각 행의 최대 글자 수는 70자로 제한한다.
- 첫 번째 행에 'feat:', 'fix:' 등의 접두사 사용을 불허한다.
- 세부 내용의 정리가 필요한 경우 불렛 포인트를 사용하며 기호는 `-`을 사용한다.
